ListBase The ListBase is a abstraction resource for datatable lists, providing a default structure and functionalities for listing data. It serves as a base component that can be extended to create specific list implementations, such as user lists, product lists, etc. Installation Use the Koala CLI to generate a new list-base component. API

Attributes

  • currentPage: Signal that holds the current page number of the list. It can be used for pagination purposes and to keep track of the user's position within the list.
  • pageSize: Signal that holds the number of items per page in the list. It can be used for pagination purposes and to control the number of items displayed on each page.
  • totalItems: Signal that holds the total number of items in the list. It can be used for pagination purposes and to display the total count of items.
  • orderedBy: Signal that holds the current ordering of the list. It can be used to sort the list based on different criteria.
  • filter: Signal that holds the current filter applied to the list. It can be used to filter the list based on different criteria.
  • skeletonItems: Computed signal that holds the number of skeleton items to display while the list is loading. It can be used to provide a visual indication of loading state.
  • defaultList: Default list response to use in datalist resourceRef.
  • filterPayload: Payload used for filtering the list. It can be used to apply specific filter criteria to the list.
  • filterParams: Parameters used for filtering the list. It can be used to apply specific filter criteria to the list.
  • datalist: Abstract datalist resource reference that should be implemented in the extending component. It serves as a reference to the data source for the list and provides methods for fetching and manipulating the list data.
  • reload: Signal that triggers a reload of the list. It can be used to refresh the list data.

Methods

  • reloadList: Method that sets the reload signal to true, propagating the reload event to the datatable component.

Filter Definitions

  • You can pass filter definitions in the super constructor of the extending component. These definitions will be used to generate the filter form and to apply the filters to the list.
